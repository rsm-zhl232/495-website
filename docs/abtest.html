A/B Testing a Call to Action

Simulating Key Ideas from Classical Frequentist Statistics

Author

Your Name

Published

April 14, 2026

Introduction

When someone lands on a website, even a small change in wording can affect whether they take action. Suppose my homepage includes a place where visitors can sign up for a newsletter, and I want to know which message encourages more sign-ups. I will compare two possible calls to action:

  • CTA A: “Sign up for our newsletter here!”
  • CTA B: “Stay up to date by signing up!”

A standard way to evaluate this kind of design choice is an A/B test. Visitors are randomly assigned to see one of the two versions, and then I compare the sign-up rates across the two groups. Random assignment matters because it helps make the groups comparable, so any systematic difference in outcomes can be attributed to the CTA rather than to differences in the visitors themselves.

In this post, I use a simulated A/B test to explain several core ideas from classical frequentist statistics: the law of large numbers, bootstrap standard errors, the central limit theorem, hypothesis testing, the regression interpretation of a two-sample test, and why “peeking” at results can create misleading findings.

The A/B Test as a Statistical Problem

Each visitor either signs up or does not sign up, so each individual outcome can be modeled as a Bernoulli random variable. If a visitor sees CTA A, their sign-up probability is \(\pi_A\). If a visitor sees CTA B, their sign-up probability is \(\pi_B\).

That means each observation is either:

  • \(1\) if the visitor signs up
  • \(0\) if the visitor does not sign up

The parameter I care about is the difference in sign-up rates:

\[ \theta = \pi_A - \pi_B \]

If \(\theta > 0\), CTA A performs better. If \(\theta < 0\), CTA B performs better. If \(\theta = 0\), the two CTAs perform equally well.

A natural estimator for this quantity is the difference in sample means:

\[ \hat{\theta} = \bar{X}_A - \bar{X}_B \]

Because the data are binary, each sample mean is just a sample proportion. So \(\bar{X}_A = \hat{\pi}_A\) and \(\bar{X}_B = \hat{\pi}_B\).

Simulating Data

In a real A/B test, we would not know the true values of \(\pi_A\) and \(\pi_B\). We would only observe data and try to infer the underlying sign-up rates. For this exercise, though, it is useful to set the truth ourselves so that we can study how our estimator behaves when we know the answer in advance.

I will assume:

  • \(\pi_A = 0.22\)
  • \(\pi_B = 0.18\)

So the true difference is:

\[ \theta = 0.22 - 0.18 = 0.04 \]

The next chunk simulates 1,000 visitors in each group.

n_A = 1000
n_B = 1000
pi_A = 0.22
pi_B = 0.18
theta_true = pi_A - pi_B

A = np.random.binomial(1, pi_A, n_A)
B = np.random.binomial(1, pi_B, n_B)

df = pd.DataFrame({
    "group": ["A"] * n_A + ["B"] * n_B,
    "signup": np.concatenate([A, B])
})

summary_table = pd.DataFrame({
    "Group": ["A", "B"],
    "Sample size": [n_A, n_B],
    "True signup rate": [pi_A, pi_B],
    "Observed signup rate": [A.mean(), B.mean()]
})

summary_table["True signup rate"] = summary_table["True signup rate"].map(lambda x: f"{x:.3f}")
summary_table["Observed signup rate"] = summary_table["Observed signup rate"].map(lambda x: f"{x:.3f}")

Even though the true rates are 0.22 and 0.18, the observed proportions in any one sample will usually not match those values exactly. That sampling variation is the main reason statistical inference is necessary.

The Law of Large Numbers

The Law of Large Numbers (LLN) says that as the sample size grows, the sample mean converges to the population mean. In this setting, that means the average outcome in group A should get closer to \(\pi_A\), the average outcome in group B should get closer to \(\pi_B\), and therefore the difference in sample means \(\hat{\theta}\) should get closer to the true difference \(\theta\) as we collect more data.

To make this concrete, I compute the element-wise differences between the simulated A and B outcomes, then track the cumulative average of those differences.

diffs = A - B
running_mean = np.cumsum(diffs) / np.arange(1, len(diffs) + 1)

At the beginning of the sample, the cumulative average jumps around a lot because each new observation has a large influence on the overall mean. As more observations accumulate, those random fluctuations become less important and the running mean settles down near the true value of 0.04. That is the practical value of the law of large numbers: it explains why larger experiments produce more stable estimates.

Bootstrap Standard Errors

Once we have a point estimate, the next question is how much uncertainty surrounds it. A point estimate alone does not tell us whether the observed difference is precise or noisy.

One flexible way to measure precision is the bootstrap. The idea is simple: repeatedly resample with replacement from the observed data, recompute the statistic on each resample, and use the spread of those resampled statistics to estimate the standard error.

Here I generate 1,000 bootstrap resamples for each group and compute a bootstrapped estimate of \(\theta\) for each one.

theta_hat = A.mean() - B.mean()

n_boot = 1000
boot_thetas = np.empty(n_boot)

for i in range(n_boot):
    A_star = np.random.choice(A, size=n_A, replace=True)
    B_star = np.random.choice(B, size=n_B, replace=True)
    boot_thetas[i] = A_star.mean() - B_star.mean()

boot_se = boot_thetas.std(ddof=1)

pi_hat_A = A.mean()
pi_hat_B = B.mean()

analytic_se = np.sqrt(
    pi_hat_A * (1 - pi_hat_A) / n_A +
    pi_hat_B * (1 - pi_hat_B) / n_B
)

ci_lower = theta_hat - 1.96 * boot_se
ci_upper = theta_hat + 1.96 * boot_se

bootstrap_results = pd.DataFrame({
    "Quantity": [
        "Estimated difference (theta_hat)",
        "Bootstrap standard error",
        "Analytical standard error",
        "95% CI lower bound",
        "95% CI upper bound"
    ],
    "Value": [theta_hat, boot_se, analytic_se, ci_lower, ci_upper]
})

bootstrap_results["Value"] = bootstrap_results["Value"].map(lambda x: f"{x:.4f}")
bootstrap_results
Quantity Value
0 Estimated difference (theta_hat) 0.0260
1 Bootstrap standard error 0.0175
2 Analytical standard error 0.0175
3 95% CI lower bound -0.0083
4 95% CI upper bound 0.0603

The bootstrap standard error and the analytical standard error should be very close, and in this simulation they usually are. That is reassuring because it tells us that the resampling approach is capturing the same uncertainty that the closed-form Bernoulli formula does.

Using the bootstrap standard error, I construct a 95% confidence interval as \(\hat{\theta} \pm 1.96 \times SE_{boot}\). Interpreted informally, this interval gives a plausible range of values for the true difference in sign-up rates. If the interval excludes zero, that suggests the data are more consistent with a real CTA effect than with no difference at all.

The Central Limit Theorem

The Central Limit Theorem (CLT) says that, under broad conditions, the sampling distribution of a sample mean becomes approximately Normal as the sample size grows. Because my estimator is the difference of two sample means, the same idea applies here: even though the underlying data are Bernoulli rather than Normal, the distribution of \(\hat{\theta}\) becomes increasingly bell-shaped when the sample size is large.

To illustrate that, I simulate the estimator 1,000 times at four different sample sizes.

sample_sizes = [25, 50, 100, 500]
clt_results = {}

for n in sample_sizes:
    estimates = np.empty(1000)
    for i in range(1000):
        A_n = np.random.binomial(1, pi_A, n)
        B_n = np.random.binomial(1, pi_B, n)
        estimates[i] = A_n.mean() - B_n.mean()
    clt_results[n] = estimates

With only 25 observations per group, the histogram is still fairly rough and discrete-looking. By the time the sample size reaches 100 or 500 per group, the distribution is much smoother and more symmetric. This is exactly the pattern predicted by the CLT. The theorem is important because it justifies using Normal-based confidence intervals and hypothesis tests in large samples, even when the raw data are binary.

Hypothesis Testing

A hypothesis test starts by specifying a null hypothesis and an alternative hypothesis. Here the natural null is:

\[ H_0: \theta = 0 \]

which says that the two CTAs have the same sign-up rate. The alternative is:

\[ H_1: \theta \neq 0 \]

which says that the CTAs differ.

To test this, I standardize the observed estimate:

\[ z = \frac{\hat{\theta} - 0}{SE(\hat{\theta})} \]

Why do this? Because the CLT tells us that for large samples, \(\hat{\theta}\) is approximately Normal. Under the null, its mean is 0. Dividing by its standard error puts the statistic onto the standard Normal scale. Once that is done, I can compute a p-value using the familiar \(N(0,1)\) distribution.

This is often loosely called a “t-test,” but that label is not quite exact here. The classical t-test comes from Normal data with an estimated variance, which produces an exact t-distribution. In this setting, the observations are Bernoulli rather than Normal, so the CLT gives us only an approximate Normal result. In large samples, though, the standard Normal and the t-distribution are so similar that the practical difference is usually small.

z_stat = theta_hat / analytic_se
p_value = 2 * (1 - norm.cdf(abs(z_stat)))

hypothesis_results = pd.DataFrame({
    "Statistic": ["Estimated difference", "Standard error", "z statistic", "Two-sided p-value"],
    "Value": [theta_hat, analytic_se, z_stat, p_value]
})

hypothesis_results["Value"] = hypothesis_results["Value"].map(lambda x: f"{x:.4f}")
hypothesis_results
Statistic Value
0 Estimated difference 0.0260
1 Standard error 0.0175
2 z statistic 1.4858
3 Two-sided p-value 0.1373

If the p-value is below a threshold such as 0.05, I reject the null hypothesis and conclude that the two CTAs likely differ in performance. If it is larger, then the observed difference is not strong enough to rule out ordinary sampling variation.

The T-Test as a Regression

The same comparison can also be written as a regression. Stack the observations into one dataset and define:

  • \(Y_i = 1\) if visitor \(i\) signs up, and \(0\) otherwise
  • \(D_i = 1\) if visitor \(i\) sees CTA A, and \(0\) if visitor \(i\) sees CTA B

Then fit:

\[ Y_i = \beta_0 + \beta_1 D_i + \varepsilon_i \]

This regression has a very intuitive interpretation:

  1. \(\beta_0\) is the mean of group B, so it estimates \(\pi_B\)
  2. \(\beta_0 + \beta_1\) is the mean of group A, so it estimates \(\pi_A\)
  3. Therefore \(\beta_1 = \pi_A - \pi_B = \theta\)

So the coefficient on the treatment indicator should match the difference in sample means from the earlier sections.

df_reg = df.copy()
df_reg["D"] = (df_reg["group"] == "A").astype(int)

X = sm.add_constant(df_reg["D"])
model = sm.OLS(df_reg["signup"], X).fit()

beta1 = model.params["D"]
beta1_se = model.bse["D"]
beta1_t = model.tvalues["D"]
beta1_p = model.pvalues["D"]

regression_results = pd.DataFrame({
    "Quantity": ["beta_1 estimate", "Standard error", "t statistic", "p-value"],
    "Regression": [beta1, beta1_se, beta1_t, beta1_p],
    "Two-sample approach": [theta_hat, analytic_se, z_stat, p_value]
})

regression_results["Regression"] = regression_results["Regression"].map(lambda x: f"{x:.4f}")
regression_results["Two-sample approach"] = regression_results["Two-sample approach"].map(lambda x: f"{x:.4f}")
regression_results
Quantity Regression Two-sample approach

1 Standard error 0.0175 0.0175

3 p-value 0.1377 0.1373

The coefficient estimate is numerically the same as the difference in means, as expected. The test statistics and p-values are also nearly identical. This equivalence matters because it shows that regression is not a completely different tool. In simple experiments, it is just another way to express the same comparison. More importantly, regression scales naturally: if I want to add controls, interactions, or multiple treatment arms later, I can do so within the same framework.

The Problem with Peeking

Now suppose my boss is impatient and wants to peek at the results every time another 100 visitors per group have been observed. That means testing after 100, 200, 300, …, up to 1,000 observations per group. If any one of those tests is significant at the 0.05 level, she wants to stop early and declare a winner.

This is a problem under classical frequentist inference. A single hypothesis test at \(\alpha = 0.05\) has a 5% chance of a false positive when the null is true. But repeated looks at accumulating data create repeated opportunities to get “lucky” noise. The overall probability of seeing at least one statistically significant result becomes much larger than 5%.

To show that, I simulate a world where the null is true: \(\pi_A = \pi_B = 0.20\).

np.random.seed(496)

n_total = 1000
peek_points = np.arange(100, 1001, 100)
n_experiments = 10000
pi_null = 0.20

any_sig = np.zeros(n_experiments, dtype=bool)

for exp in range(n_experiments):
    A_null = np.random.binomial(1, pi_null, n_total)
    B_null = np.random.binomial(1, pi_null, n_total)

    sig_found = False

    for n in peek_points:
        A_n = A_null[:n]
        B_n = B_null[:n]

        theta_n = A_n.mean() - B_n.mean()
        se_n = np.sqrt(
            A_n.mean() * (1 - A_n.mean()) / n +
            B_n.mean() * (1 - B_n.mean()) / n
        )

        if se_n == 0:
            continue

        z_n = theta_n / se_n
        p_n = 2 * (1 - norm.cdf(abs(z_n)))

        if p_n < 0.05:
            sig_found = True
            break

    any_sig[exp] = sig_found

false_positive_rate = any_sig.mean()

peeking_results = pd.DataFrame({
    "Procedure": ["Single test at n = 1000", "Peeking every 100 observations"],
    "False positive rate": [0.05, false_positive_rate]
})

peeking_results["False positive rate"] = peeking_results["False positive rate"].map(lambda x: f"{x:.4f}")
peeking_results
Procedure False positive rate
0 Single test at n = 1000 0.0500
1 Peeking every 100 observations 0.2014

The simulation usually shows that the false positive rate under repeated peeking is much larger than 0.05. In other words, if I repeatedly check the data and stop whenever I happen to cross the significance threshold, I dramatically increase my chance of falsely declaring a winner.

That is why A/B tests need a pre-specified stopping rule or a formal sequential testing procedure. Without one, “significance” can become a product of impatience rather than evidence.

Conclusion

This A/B testing example ties together several foundational ideas in frequentist statistics.

  • The Law of Large Numbers explains why the difference in sample means stabilizes as the experiment grows.
  • The bootstrap provides a practical way to estimate uncertainty.
  • The Central Limit Theorem justifies Normal-based inference for large samples.
  • Hypothesis testing turns the estimate into a formal decision rule.
  • The same comparison can be expressed as a regression, showing how experimental analysis fits into a broader modeling framework.
  • Finally, peeking demonstrates that statistical procedures depend not just on formulas, but also on disciplined experimental design.

Even in a simple A/B test, the language of classical statistics gives us a rigorous way to reason about evidence, uncertainty, and decision-making.
